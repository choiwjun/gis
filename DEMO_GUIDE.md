## 📋 시연 시나리오

### 1️⃣ 로그인 및 대시보드 (30초)

1. 데모 URL 접속
2. 자동으로 표시되는 로그인 폼에서:
   - Email: `admin@example.com`
   - Password: `admin123`
3. **Login** 버튼 클릭
4. 좌측에 데이터셋 목록이 표시되는 메인 화면 진입

**확인 포인트**:
- ✅ 토스트 알림: "Login successful" (녹색)
- ✅ 좌측 사이드바에 사용자 이름 표시: "Welcome, Admin User"
- ✅ 4개의 데이터셋 목록 표시

---

### 2️⃣ 데이터셋 선택 및 지도 표시 (1분)

**시연할 데이터셋**: "서울 주요 관광지 2025"

1. 좌측 사이드바에서 **"서울 주요 관광지 2025"** 클릭
2. 지도에 자동으로 포인트들이 표시됨
3. 지도가 자동으로 서울 지역으로 이동 (FitBounds)

**확인 포인트**:
- ✅ 31개의 포인트가 서울 지역에 표시
- ✅ 선택된 데이터셋이 파란색 배경으로 하이라이트
- ✅ 토스트 알림: "Dataset loaded on map"
- ✅ 포인트들이 파란색 원으로 표시 (중심은 흰색 테두리)

---

### 3️⃣ 클러스터링 기능 시연 (1분)

1. 지도를 **축소(Zoom Out)** 하여 전체 뷰 확인
2. 포인트들이 자동으로 **클러스터**로 그룹화되는 것 확인
   - 클러스터 색상:
     - 10개 미만: 청록색 (#51bbd6)
     - 10-30개: 노란색 (#f1f075)
     - 30개 이상: 분홍색 (#f28cb1)
3. **클러스터를 클릭**하면 자동으로 확대(Zoom In)
4. 개별 포인트가 보일 때까지 반복

**확인 포인트**:
- ✅ 줌 레벨에 따라 클러스터 크기와 색상 변화
- ✅ 클러스터 내 포인트 개수 표시
- ✅ 클러스터 클릭 시 부드러운 줌 애니메이션
- ✅ 줌 인 시 클러스터가 개별 포인트로 분리

---

### 4️⃣ 피처 상세 정보 (1분)

1. 개별 포인트 클릭 (예: **"경복궁"** 또는 **"강남역"**)
2. 우측에 디테일 패널이 슬라이드 인 애니메이션으로 표시
3. 선택된 포인트가 **금색 원**으로 하이라이트
4. 패널에서 다음 정보 확인:
   - 이름 (name)
   - 카테고리 (category)
   - 타입 (type)
   - 점수 (score)
   - 주소 (address)
   - 설명 (description)
   - 좌표 (coordinates)

**확인 포인트**:
- ✅ 우측 패널 슬라이드 인 애니메이션 (300ms)
- ✅ 선택된 포인트에 금색(#FFD700) + 주황색 테두리 표시
- ✅ 속성 정보가 깔끔하게 정리되어 표시
- ✅ X 버튼으로 패널 닫기 가능
- ✅ **ESC 키**로도 패널 닫기 가능

---

### 5️⃣ 베이스맵 전환 (30초)

1. 좌측 상단의 **베이스맵 컨트롤** 확인
2. **"Satellite"** 버튼 클릭
3. 지도가 위성 이미지로 전환
4. **"Standard"** 버튼을 클릭하여 다시 일반 지도로 전환

**확인 포인트**:
- ✅ 베이스맵이 즉시 전환
- ✅ 토스트 알림: "Basemap switched to satellite/standard"
- ✅ 선택된 버튼이 파란색으로 하이라이트
- ✅ 포인트와 클러스터는 계속 표시

---

### 6️⃣ 검색 기능 (1분)

1. 우측 상단의 **검색창**에 키워드 입력:
   - 예: **"강남"** 또는 **"관광"** 또는 **"문화재"**
2. Enter 키 또는 🔍 버튼 클릭
3. 검색 결과로 지도가 자동 이동 (FlyTo 애니메이션 1.5초)
4. 첫 번째 검색 결과로 줌 레벨 14로 이동

**검색 예시**:
- **"경복궁"**: 경복궁으로 이동
- **"교통"**: 교통 카테고리 (서울역, 강남역 등)
- **"관광"**: 관광 카테고리 (경복궁, 롯데월드타워 등)

**확인 포인트**:
- ✅ 검색 결과 개수 토스트 표시: "Found X results"
- ✅ 부드러운 FlyTo 애니메이션
- ✅ 검색 결과가 없으면: "No results found"
- ✅ 데이터셋 미선택 시: "Please select a dataset first"

---

### 7️⃣ 데이터 업로드 (1분 30초)

1. 좌측 사이드바 상단의 **"Upload Dataset"** 버튼 클릭
2. 모달 다이얼로그가 표시됨
3. 업로드 폼 작성:
   - **Dataset Name**: "테스트 데이터셋"
   - **Type**: GeoJSON 선택
   - **File**: `demo_data.geojson` 선택 (프로젝트 루트에 있음)
4. **"Upload"** 버튼 클릭
5. 프로그레스바가 0% → 100%로 진행
6. 업로드 완료 후 데이터셋 목록에 자동 추가

**확인 포인트**:
- ✅ 업로드 중 프로그레스바 애니메이션
- ✅ 성공 토스트: "Dataset uploaded successfully"
- ✅ 모달이 자동으로 닫힘
- ✅ 좌측 사이드바에 새 데이터셋 추가
- ✅ ESC 키로 모달 닫기 가능

**CSV 업로드 예시** (선택사항):
- Type: CSV 선택
- File: `demo_cafes.csv`
- CSV는 자동으로 lat/lon 컬럼을 감지하여 GeoJSON으로 변환

---

### 8️⃣ 사용자 관리 (Admin 전용, 1분)

1. 좌측 하단의 **"User Management"** 버튼 클릭
2. 사용자 목록 모달 표시
3. 현재 등록된 사용자 확인:
   - Admin User (admin)
   - Editor User (editor)
   - Viewer User (viewer)
4. 각 사용자의 역할이 색상별 뱃지로 표시:
   - 🔴 Admin: 빨간색
   - 🔵 Editor: 파란색
   - ⚪ Viewer: 회색
5. 편집 버튼(✏️) 클릭 가능 (실제 편집 UI는 placeholder)

**확인 포인트**:
- ✅ Admin만 접근 가능 (다른 역할은 경고 토스트)
- ✅ 사용자 목록이 페이지네이션과 함께 표시
- ✅ 역할별 색상 구분
- ✅ ESC 키로 모달 닫기

---

### 9️⃣ 반응형 레이아웃 (30초)

**데스크톱 레이아웃**:
- 3칼럼 구조: 데이터셋 목록 | 지도 | 디테일 패널
- 좌측: 320px (w-80)
- 중앙: 나머지 공간 (flex-1)
- 우측: 384px (w-96)

**모바일 레이아웃** (768px 이하):
1. 브라우저 창을 좁게 조정
2. 디테일 패널이 하단에서 슬라이드 업
3. 데이터셋 사이드바가 상단으로 이동
4. 터치 친화적 UI로 전환

**확인 포인트**:
- ✅ 반응형 breakpoint: 768px
- ✅ 모바일에서 디테일 패널이 하단 고정 (max-height: 50vh)
- ✅ 스크롤 가능한 컨텐츠
- ✅ 터치 최적화

## 🎯 데모 데이터 상세

### 1. 서울 주요 관광지 2025 (GeoJSON)
**31개 장소 포함**:

| 카테고리 | 장소 예시 | 점수 |
|---------|---------|------|
| 교통 | 서울역, 강남역, 김포공항 | 90-98 |
| 관광 | 경복궁, 롯데월드타워, 명동성당 | 85-94 |
| 문화 | 홍대입구역, 이태원, 건대입구역 | 83-90 |
| 상업 | 코엑스, 명동거리, 남대문시장 | 80-93 |
| 공원 | 여의도공원, 한강공원, 석촌호수 | 82-86 |
| 정치 | 청와대, 광화문광장 | 89-91 |

**속성 정보**:
- `name`: 장소명 (한글)
- `category`: 카테고리 (교통/관광/문화/상업/공원/정치)
- `type`: 세부 타입
- `score`: 인기도 점수 (75-98)
- `address`: 주소
- `description`: 설명

### 2. 서울 인기 카페 리스트 (CSV)
**15개 카페 체인점**:
- 스타벅스, 투썸플레이스, 이디야, 할리스
- 커피빈, 엔제리너스, 블루보틀, 폴바셋
- 메가커피, 컴포즈커피, 빽다방, 탐앤탐스 등

**속성 정보**:
- `name`: 카페명
- `lat`, `lon`: 좌표
- `category`: 카페
- `rating`: 평점 (3.9-4.8)
- `price_range`: 가격대 (저/중/고)
- `description`: 위치 설명

## 🐛 알려진 제약사항

1. **CSV 업로드 이슈**:
   - CSV 파일의 lat/lon 컬럼이 정확히 감지되지 않을 수 있음
   - 헤더는 `lat`, `lon` 또는 `latitude`, `longitude` 사용 권장

2. **Shapefile 지원**:
   - Shapefile은 placeholder만 구현
   - 실제 파싱은 서버사이드 라이브러리 필요

3. **대용량 데이터**:
   - 현재 1,000개 피처까지만 저장 (코드에서 제한)
   - 더 많은 데이터는 페이지네이션 필요
